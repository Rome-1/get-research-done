# LLM Mechanistic Interpretability Error Catalog — Core Classes (#1-25)

Language models make characteristic errors when assisting with mechanistic interpretability research. These differ from physics errors: instead of sign mistakes and wrong identities, LLMs confuse patching directions, hallucinate feature interpretations, misattribute causal roles, and make systematic mistakes in experimental design. This catalog documents the most common error classes with detection strategies.

Consult this catalog before trusting any LLM-generated mech-interp analysis. Every error class below has been observed in production.

## Error Classes

| # | Error Class | Description | Detection Strategy | Example |
|---|---|---|---|---|
| 1 | **Confusing denoising and noising** | LLMs swap the direction of activation patching: describing denoising (patch clean into corrupted run) as noising (patch corrupted into clean run) or vice versa. This inverts the interpretation: sufficiency claims become necessity claims and vice versa. | Check which run serves as the base and which provides the patched activations. Denoising: base=corrupted, patch=clean (tests sufficiency). Noising: base=clean, patch=corrupted (tests necessity). Verify the description matches the code. | LLM describes "patching clean activations into the corrupted run to test which components are necessary" — but this tests sufficiency, not necessity. Necessary components are found by noising (corrupting clean activations). |
| 2 | **Wrong residual stream composition** | LLMs misunderstand how transformer components compose through the residual stream. They treat head outputs as independent when they actually sum into the residual stream, or they forget that an attention head at layer L reads the residual stream (sum of all prior components), not just the previous layer's output. | Trace the data flow explicitly: the input to layer L's attention is the residual stream = embed + sum of all attention and MLP outputs from layers 0 to L-1. Each head reads from this cumulative sum, not from a single prior component. Verify that any claim about "head A reads head B's output" accounts for all other components also contributing to the residual stream. | LLM claims "head 9.1 reads the output of head 7.3" when actually head 9.1 reads the full residual stream, which includes contributions from all 8×12 = 96 prior attention heads plus 8 MLPs plus the embedding. Head 7.3's contribution may be a small fraction of this input. |
| 3 | **Hallucinating feature interpretations** | LLMs generate plausible-sounding but incorrect interpretations of SAE features or neurons based on insufficient evidence. They confidently label a feature based on 2-3 cherry-picked examples and ignore counterexamples. | Verify interpretations against ≥20 max-activating examples AND ≥20 random activating examples. Check for counterexamples: inputs matching the description that don't activate, and inputs not matching that do activate. Use automated interpretability scores when available. | LLM sees feature top-3 activations on "cat", "kitten", "tabby" and labels it "cat-related." But the full top-20 includes "hat", "mat", "bat" — the feature responds to short words ending in "at", not cats specifically. |
| 4 | **Treating probing accuracy as causal evidence** | LLMs conclude that because a linear probe achieves high accuracy at layer L, the model "knows" or "uses" that information at layer L. Probing is correlational — information may be present but unused by the model. | Pair every probing result with causal evidence: does ablating layer L impair the model's use of this information? If the probe works but ablation doesn't matter, the information is an unused byproduct, not a functional representation. Run control probes on random labels to establish baseline. | LLM finds 85% probing accuracy for sentiment at layer 4 and concludes "the model processes sentiment at layer 4." But causal tracing shows sentiment processing is at layer 10 — layer 4 just happens to encode word-level features that correlate with sentiment without being used for sentiment-dependent predictions. |
| 5 | **Wrong ablation baseline claims** | LLMs claim to use "mean ablation" but describe zero ablation, or claim "resample ablation" but use a fixed corrupted input rather than sampling from a distribution. The choice of baseline critically affects results. | Verify the exact operation: zero ablation sets activations to 0; mean ablation replaces with the average over a reference dataset; resample ablation replaces with activations from a different input (sampled each time). Check the code matches the description. | LLM describes "mean ablation" but the code sets `hook_result = torch.zeros_like(hook_result)` — that's zero ablation. Mean ablation requires precomputing `mean_activation = activations.mean(dim=0)` over a reference dataset. |
| 6 | **Layer-off-by-one errors** | LLMs confuse pre- and post-layer hook points, mis-index layers (0-indexed vs 1-indexed), or place interventions at the wrong point in the residual stream (before vs after attention, before vs after MLP, before vs after layer norm). | Map every hook point to the exact position in the forward pass. In a standard pre-norm transformer: resid_pre → layer_norm → attention → resid_mid → layer_norm → MLP → resid_post. Verify that "layer 6 attention output" means the attention head outputs at layer 6, not layer 5 or 7. Check framework-specific naming conventions. | LLM patches "the residual stream after layer 6" (resid_post for layer 6) when they meant "the input to layer 6's attention" (resid_pre for layer 6). In pre-norm architectures, these differ by the layer 6 MLP output, which can be the critical component. |
| 7 | **Ignoring attention head QKV decomposition** | LLMs treat attention heads as atomic units when the query, key, value, and output matrices serve distinct roles. A head can be important for its OV (output-value) circuit while its QK (query-key) attention pattern is irrelevant to the task, or vice versa. | When a head is identified as important, decompose: (1) Is the attention pattern (QK) or the value processing (OV) responsible? (2) Does the head read from specific subspaces (via the Q and K projections)? (3) Does it write to specific subspaces (via the OV circuit)? Path patching through QKV components tests this. | LLM says "head 9.1 attends to the subject token and that's why it's important." But the attention pattern is incidental — what matters is the OV circuit that maps the subject token's residual stream to the correct output direction. A different head with the same OV circuit but different attention pattern would be equally effective. |
| 8 | **Distribution shift from ablation** | LLMs perform ablation experiments without considering that the ablated activations may be out-of-distribution for downstream components. Zero ablation at an early layer pushes activations into a regime the model has never seen, causing cascading failures that are not informative about the ablated component's function. | Use mean ablation or resample ablation instead of zero ablation for causal claims. For zero ablation results, verify the conclusion holds under mean ablation. Report both when possible. Measure whether the post-ablation activations at later layers are within the normal range (e.g., check the norm of the residual stream after ablation). | LLM zero-ablates all attention heads in layer 2 and reports "layer 2 attention is critical — loss increases 50x." But mean-ablating the same heads increases loss by only 1.3x. The zero ablation result reflects distribution shift from having abnormally-zeroed components in the residual stream, not the heads' actual importance. |
| 9 | **Confusing superposition and polysemanticity** | LLMs use "superposition" and "polysemanticity" interchangeably, but they are distinct phenomena. Superposition: the model represents more features than it has dimensions by using non-orthogonal directions. Polysemanticity: a single neuron activates for multiple unrelated concepts. Superposition causes polysemanticity but not all polysemanticity comes from superposition (it can also arise from incidental feature correlation in training data). | Verify: superposition refers to the representational geometry (more features than dimensions, non-orthogonal encoding). Polysemanticity refers to an observable property of individual neurons (multiple activation patterns). A claim about superposition should reference the dimensionality and interference between features; a claim about polysemanticity should reference a specific unit's activation pattern. | LLM writes "we observe superposition because neuron 42 activates for both 'dog' and 'cat' contexts." This is polysemanticity (one neuron, multiple patterns), not superposition (more features than dimensions). Superposition is the geometric explanation for why polysemanticity occurs, not the observed phenomenon itself. |
| 10 | **Incorrect logit attribution decomposition** | LLMs make errors when decomposing the logit output into contributions from individual components. The residual stream at the final position sums contributions from all heads and MLPs, and each contribution's effect on the logit is its dot product with the unembedding vector. LLMs often forget to include the bias term, confuse logits with log-probabilities, or mis-apply the unembedding matrix. | Verify the decomposition: logit(token) = (sum of all component outputs at final position) · W_U[token] + b_U[token]. Check that the sum of individual attributions equals the total logit difference (within numerical precision). If they don't sum correctly, a term is missing. The unembedding matrix W_U maps from residual stream to vocabulary — verify its orientation (rows vs columns). | LLM decomposes the logit into head contributions but forgets the MLP contributions and the embedding bias. The individual head attributions sum to 3.2 but the actual logit difference is 7.8. The missing 4.6 is from MLPs and biases, which are silently omitted. |
| 11 | **Wrong causal tracing corruption strength** | LLMs use arbitrary noise levels for causal tracing without calibrating to the model. Too little noise and the model still predicts correctly (trace is uninterpretable); too much noise and the model is completely broken (trace shows everything is important). | Before running causal tracing, verify: (1) the corrupted model's probability of the correct answer is <10% (corruption is effective), (2) the corrupted model still produces coherent probabilities — not NaN or uniform (corruption is not too strong). Report the noise level used and the corrupted-model probability. | LLM runs causal tracing with noise σ = 0.1 (too weak for GPT-2 scale). The model still predicts "Paris" with 70% probability after corruption. The causal trace shows large restoration effects at every position because even small restorations push the already-high probability slightly higher. No localization conclusion is valid. |
| 12 | **Confusing attention scores with information flow** | LLMs interpret high attention scores as evidence of information flow between positions. But attention patterns alone don't determine what information moves — that depends on the value vectors and the OV circuit. A head can attend strongly to a position and copy nothing useful from it, or attend weakly and copy critical information. | Never conclude "information flows from position A to position B" based solely on attention patterns. Always examine: (1) what is in the value vector at position A, (2) what the OV circuit does to that value, and (3) whether the head's contribution to the residual stream actually affects downstream computation. Use activation patching to test causal information flow. | LLM finds head 5.3 attends heavily to the first token in every sequence and concludes it's an "information aggregation head." But the first token position contains only the BOS embedding, and the OV circuit projects this to a near-zero vector. The head does nothing despite attending strongly. |
| 13 | **Incorrect SAE reconstruction fidelity claims** | LLMs report SAE quality metrics incorrectly: confusing MSE loss (reconstruction of activations) with loss recovered (how much the model's behavior is preserved when SAE activations replace originals). An SAE can have low MSE but poor loss recovered if it smooths out critical high-frequency features. | Always report both metrics separately: (1) MSE = mean squared error between original and reconstructed activations. (2) Loss recovered = 1 - (CE_loss_with_SAE - CE_loss_original) / (CE_loss_zero_ablation - CE_loss_original). These measure different things. Verify that "95% reconstruction" specifies which metric. | LLM reports "the SAE achieves 95% reconstruction fidelity" based on explained variance (R²) of activations. But loss recovered is only 78% — the SAE misses subtle but behaviorally important activation patterns. The 95% number is misleading for downstream use. |
| 14 | **Ignoring positional effects in patching** | LLMs patch activations between prompts of different lengths without accounting for positional encoding differences. If clean and corrupted prompts have different token counts, patching at "position 5" refers to different absolute positions, and the positional encoding mismatch corrupts the results. | Ensure clean and corrupted prompts have identical tokenization length. If they must differ in length, patch by relative position (e.g., "the subject token" rather than "position 5") and verify that positional encoding differences are not driving the results. Test with prompts of matched length as a control. | LLM patches activations from a 10-token clean prompt into a 15-token corrupted prompt at "the same position." But position 5 in the clean prompt corresponds to a different word than position 5 in the corrupted prompt due to different tokenization. The patching result reflects position mismatch, not component importance. |
| 15 | **Confusing feature activation with feature importance** | LLMs equate how strongly a feature activates (its activation magnitude) with how important it is for model behavior. A feature can activate strongly but have a small decoder norm (low influence on the residual stream) or point in an irrelevant direction. Conversely, a moderately-activating feature with a large decoder norm pointing toward the output logit can be highly influential. | Compute feature attribution = activation × decoder_direction · W_U[target_token], not just activation magnitude. This accounts for both how strongly the feature fires and how much its firing affects the output. Compare: does the ranking by activation magnitude agree with the ranking by attribution? If not, report the attribution ranking. | LLM identifies "the most important feature for this prediction" as the one with the highest activation (value = 15.2). But this feature's decoder direction is nearly orthogonal to the target token's unembedding vector, giving it an attribution of 0.3. Feature #2847 activates at 3.1 but has attribution 8.7 because its decoder direction aligns with the output. |
| 16 | **Wrong number of effective features** | LLMs confuse the SAE dictionary size (total number of learned features) with the L0 norm (average number of active features per input) or with the number of interpretable features. A 32K-feature SAE may have L0=50 (50 features active per input) and only 20K features that ever activate on a diverse corpus (the rest are dead). | Always specify which number you mean: (1) dictionary size = total feature count. (2) L0 = average active features per token. (3) alive features = features that activate on >0.01% of tokens. (4) interpretable features = alive features with interpretability score above threshold. These can differ by orders of magnitude. | LLM writes "the SAE has 32,768 features, each encoding a distinct concept." But 8,000 are dead (never activate), L0 is 50 (only 50 are active on any given input), and interpretability analysis shows only 15,000 have clear human-understandable descriptions. |
| 17 | **Not accounting for layer norm** | LLMs forget that in pre-norm transformers, activations pass through layer norm before entering attention and MLP blocks. This means the residual stream value and the value actually processed by a component differ. Patching the residual stream but measuring pre-layer-norm activations (or vice versa) introduces errors. | Map the exact transformer architecture: does it use pre-norm (layer norm before attention/MLP) or post-norm (after)? When patching, specify whether you're patching the residual stream (before layer norm) or the post-norm activation (after). SAEs trained on pre-norm activations should not be applied to post-norm activations. | LLM trains an SAE on `blocks.6.hook_resid_pre` (residual stream before layer 6's layer norm) but interprets features as "what layer 6's attention processes." The attention actually processes `blocks.6.hook_attn_in`, which is the layer-normed version. The feature directions are in the wrong space. |
| 18 | **Assuming features are orthogonal** | LLMs treat SAE features as if they span orthogonal directions, but SAE decoder vectors are NOT generally orthogonal (by design — superposition requires non-orthogonal representations). Ignoring this means feature attributions don't sum correctly and interference between features is missed. | Check: compute pairwise cosine similarities between decoder vectors of co-active features. If cosine similarity > 0.3, the features share significant variance and their attributions may interfere. When summing feature attributions, verify the sum equals the actual logit difference (interference causes deviations). | LLM decomposes a residual stream vector as "3.2 from feature A + 2.1 from feature B + 1.7 from feature C = 7.0 total." But cos(A,B) = 0.4, and the actual residual norm in the relevant direction is 5.8 due to constructive/destructive interference between the non-orthogonal feature directions. |
| 19 | **Generalizing from one model to all models** | LLMs make universal claims about "how transformers work" based on analysis of a single model (usually GPT-2 Small). Circuit structure, feature organization, and even whether specific algorithms are implemented can vary across model sizes, architectures, and training data. | Always qualify claims with the specific model studied. Distinguish "GPT-2 Small implements X" from "transformers in general implement X." Check if the finding has been replicated in at least one other model family or scale. | LLM discovers induction heads at layers 5-6 in GPT-2 Small and writes "transformers implement induction via heads in the early-mid layers." But in Llama-7B, induction-like behavior is distributed across many layers and no single pair of heads cleanly implements the pattern. |
| 20 | **Conflating MLP neurons with MLP features** | LLMs interpret individual MLP neurons as features, but MLP neurons in transformers are typically highly polysemantic (due to superposition). The meaningful features of an MLP may be non-axis-aligned directions in the neuron activation space, not individual neurons. | When interpreting MLP behavior, acknowledge that individual neurons are likely polysemantic. Use SAE features on MLP outputs rather than interpreting individual neurons. If interpreting neurons directly, always check for polysemanticity by examining max-activating examples across diverse contexts. | LLM finds MLP neuron 1847 activates on "Paris" and labels it the "Paris neuron." But the same neuron also activates on "capital", "tower", "French cuisine", and "romantic comedy" — it's a superposition of many features, not a clean "Paris detector." |
| 21 | **Wrong causal scrubbing protocol** | LLMs misapply causal scrubbing (resample ablation with structural constraints). Causal scrubbing requires that resampled activations come from inputs matched on all variables EXCEPT the one being tested. LLMs often resample from the wrong distribution or fail to match the required structural constraints. | For each component in a causal scrubbing experiment: verify (1) what information the component is hypothesized to use, (2) that the resampled activations come from inputs that differ only in the hypothesized information, (3) that structural constraints (same attention patterns, same token positions) are maintained where required. | LLM tests whether head 9.1 uses the subject's identity by resampling from random prompts. But the resampled prompts have different syntactic structures, so the test confuses "subject identity" with "syntactic role." The correct approach resamples from prompts with different subjects but identical structure. |
| 22 | **Misinterpreting negative heads** | LLMs find components with negative effects (ablating them improves performance) and label them as "harmful" or "error-inducing." Negative-contribution components often serve important roles: they may suppress incorrect alternatives, implement calibration, or counteract other components' biases. | When a component has a negative direct logit attribution (suppresses the correct answer), investigate: (1) does it suppress incorrect answers even more (net positive effect)? (2) does it interact with other components (its removal might improve one metric but break another)? (3) does it serve a calibration role (prevent overconfidence)? Never conclude a component is "bad" from negative attribution alone. | LLM finds head 10.2 has -1.5 direct logit attribution for the correct answer and labels it a "negative head that hurts performance." But head 10.2 has -3.8 attribution for the top incorrect answer — it's actually suppressing the wrong answer more than the right one, improving the logit difference by 2.3. |
| 23 | **Confusing token-level and sequence-level analysis** | LLMs mix up per-token and per-sequence metrics when analyzing model behavior. An attention head might be important for a specific token position but irrelevant for others in the same sequence. Averaging importance over all positions obscures this. | Always specify whether analysis is at the token level (which specific position) or sequence level (averaged over positions). When reporting patching results, indicate the position of intervention and the position of measurement. A head can be critical at position 15 and irrelevant at positions 0-14. | LLM reports "head 7.3 has mean patching effect of 0.02 (unimportant)" but this averages the critical effect at the indirect object position (effect = 0.8) with near-zero effects at all other positions. The head is essential for the task at the specific position where it matters. |
| 24 | **Misattributing behavior to the wrong component type** | LLMs attribute computation to attention heads when it's actually in the MLP (or vice versa) because they tested only one component type. In many tasks, attention heads route information while MLPs perform the actual computation. Missing either half gives an incomplete picture. | Always test both attention and MLP contributions at layers identified as important. For circuit discovery: include both component types in the search. For causal tracing: decompose the restoration effect by component type (attention vs MLP). Report which component type contributes what fraction. | LLM finds that patching all attention heads in layers 9-11 recovers 60% of the logit difference and concludes "the circuit is in the attention heads." But patching MLPs in the same layers recovers 40% independently. The full circuit requires both, and the MLP component is performing the key lookup/computation that the attention heads route to. |
| 25 | **Wrong SAE sparsity-fidelity tradeoff claims** | LLMs claim an SAE is "optimal" without specifying where on the sparsity-fidelity Pareto frontier it sits. Different downstream uses require different tradeoffs: circuit discovery needs high fidelity (even at lower sparsity), feature interpretation needs high sparsity (even at lower fidelity). | Always report both L0 (sparsity) and loss recovered (fidelity). When comparing SAEs, plot the full Pareto frontier. Never call an SAE "better" without specifying the metric and the use case. An SAE with L0=20 and 85% loss recovered is not comparable to one with L0=100 and 97% loss recovered without knowing the application. | LLM compares SAE-A (L0=30, loss recovered=90%) and SAE-B (L0=80, loss recovered=96%) and declares "SAE-B is better." But for circuit discovery at this layer, SAE-A's sparser features are more interpretable and the circuit contains only 5 features, while SAE-B's circuit requires 15 features for the same behavior — making the circuit harder to understand despite higher raw fidelity. |
