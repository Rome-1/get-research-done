# Publication Review Wrapper Guidance

Keep publication command wrappers thin. State the command role, include the workflow link, keep context concise, and defer stage ordering, retries, manuscript-root preflight, and schema visibility to the linked workflow. Do not restate the full publication pipeline in the wrapper.
When wrappers need publication review or revision naming, point to the shared round-artifact and artifact-gate references instead of re-embedding those rules inline.
