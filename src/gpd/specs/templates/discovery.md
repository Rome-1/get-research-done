# Discovery Template

Legacy compatibility shim for `GPD/phases/XX-name/DISCOVERY.md`.

New work should use `research.md` and write `RESEARCH.md`. Keep `DISCOVERY.md` only when you need to preserve older phase history or satisfy compatibility with pre-existing outputs.

If you are migrating a discovery-heavy flow, prefer a `depth: quick|standard|deep` `RESEARCH.md` output and treat `DISCOVERY.md` as a legacy alias, not a separate canonical artifact.
